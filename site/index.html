Who it is for

Use it when a note or vendor claim needs proof.

The point is not to replace clinical review or crown yesterday's model. The point is to make unsupported clinical claims visible, reproducible, and discussable in public: first as a one-note triage result, then as an aggregate benchmark row.

        Why this matters
        
              
        
            
        
          
        
        
        
      
        

      
        
        
        
          
        Method
        
          
        Three checks, one public scoring harness
        
          
        
            ScribeBench separates quality from safety. A note can be fluent and complete while
            still inventing something clinically meaningful, so the fabrication review is run as
            a separate adversarial judge.
          
        
        
        
        
        
          
        
            01
            
        Narrative quality
        
            
        Six physician-style dimensions: story, completeness, flow, artifacts, readability, and input fidelity.
        
          
        
          
        
            02
            
        Fabrication review
        
            
        Flags unsupported clinical content and separates dangerous invention from standard charting.
        
          
        
          
        
            03
            
        Leak detection
        
            
        Deterministic scan for raw template placeholders and internal metadata tokens.
        
          
        
        
        
      
        

      
        
        
        
          
        Use it
        
          
        Build a public evidence row
        
          
        
            Pick the run you are trying to make, generate or paste candidate notes,
            then score them with a declared judge. Smoke tests prove the plumbing.
            Powered PriMock57 runs are the only rows that support public claims.
