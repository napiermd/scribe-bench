What visitors can do

Three practical jobs, one benchmark.

This is not a consumer documentation product and it is not trying to crown a favorite model from stale launch rows. It is a public testing surface for a specific question: did the generated note preserve the clinical encounter, or did it invent something that changes care?

        Method
        
          
        Three checks, one public scoring harness
        
          
        
            ScribeBench separates quality from safety. A note can be fluent and complete while
            still inventing something clinically meaningful, so the fabrication review is run as
            a separate adversarial judge.
          
        
        
        
        
        
          
        
            01
            
        Narrative quality
        
            
        Six physician-style dimensions: story, completeness, flow, artifacts, readability, and input fidelity.
        
          
        
          
        
            02
            
        Fabrication review
        
            
        Flags unsupported clinical content and separates dangerous invention from standard charting.
        
          
        
          
        
            03
            
        Leak detection
        
            
        Deterministic scan for raw template placeholders and internal metadata tokens.
        
          
        
        
        
      
        

      
        
        
        
          
        Use it
        
          
        Run smoke tests, then a powered benchmark
        
          
        
            First generate candidate notes, then score them. The 3-case synthetic set is
            only a plumbing check. A public leaderboard row should come from the powered
            PriMock57 run with aggregate scores only.
          
        
        
        
        
        
          
        
            1
            
        Generate notes
        
            
        Run your scribe over every PriMock57 case and save a JSON array with each case ID and generated note.
        
          
        
          
        
            2
            
        Score the run
        
            
        Use a declared judge backend and model. The output summary is what belongs in the public ledger.
        
          
        
          
        
            3
            
        Open a PR
        
            
        Submit aggregate scores, judge details, generation method, and whether the system was tuned to the benchmark.
        
          
        
        
        
        
        
          
        
            
        Candidate note format
        
            
        [
  { "caseId": "PM57-d1c01", "note": "HPI: ..." },
  { "caseId": "PM57-d1c02", "note": "HPI: ..." }
]
        
          
        
          
        
            
        Generate candidate notes
        
            
        export OPENROUTER_API_KEY=...

npx tsx scripts/generate_baseline.ts \
  --gen openrouter \
  --model nvidia/nemotron-3-ultra-550b-a55b:free \
  --dataset data/primock57/cases \
  --out /tmp/openrouter_primock57_notes.json
        
          
        
          
        
            
        Powered leaderboard run
        
            
        npm install

export SCRIBEBENCH_BACKEND=baseten
export BASETEN_API_KEY=...
export SCRIBEBENCH_JUDGE_MODEL=deepseek-ai/DeepSeek-V4-Pro

npx tsx eval/run_benchmark.ts \
  --dataset data/primock57/cases \
  --candidate /tmp/openrouter_primock57_notes.json \
  --system "your-system" \
  --repeats 2 \
  --out leaderboard/_pending.json
        
          
        
          
        
            
        Smoke test only
        
            
        npx tsx eval/run_benchmark.ts \
  --dataset data/synthetic/cases \
  --candidate data/synthetic/example_candidate.json \
  --system "example-baseline" \
  --out leaderboard/_pending.json
