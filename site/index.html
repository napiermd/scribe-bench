      What this repo is

      One-note checker up front. Aggregate evidence behind it.

      ScribeBench is a public evidence loop. A single source-note pair creates a review card; a declared batch of many notes creates an aggregate row. The repo contains the site, evaluator, APIs, data, worklog, and ledger that make those artifacts repeatable.

      1. 1 Bring source plus note

        Transcript, source note, demo output, or candidate-note file.

      2. 2 Check unsupported care

        Invented results, meds, follow-up, diagnoses, orders, demographics, and leaks.

      3. 3 Use the right artifact

        QA packet, claim ask, blocker receipt, second-opinion verdict, or aggregate row.

      4. 4 Do not overclaim

        One note is triage. Current rankings need powered rows.

      Not for Patient advice, billing, clinical clearance, or crowning a current winner from old rows.

      Useful output means a reviewable artifact with evidence level and boundary, not another demo screenshot.

              Method
              
          
              Three checks, one public scoring harness
              
          
              
            ScribeBench separates quality from safety. A note can be fluent and complete while
            still inventing something clinically meaningful, so the fabrication review is run as
            a separate adversarial judge.
          
              
        
              
        
              
          
              
            01
            
              Narrative quality
              
            
              Six physician-style dimensions: story, completeness, flow, artifacts, readability, and input fidelity.
              
          
              
          
              
            02
            
              Fabrication review
              
            
              Flags unsupported clinical content and separates dangerous invention from standard charting.
              
          
              
          
              
            03
            
              Leak detection
              
            
              Deterministic scan for raw template placeholders and internal metadata tokens.
              
          
              
        
              
      
              

      
              
        
              
          
              Aggregate evidence
              
          
              Publish aggregate evidence, not another demo
              
          
              
            A citeable ScribeBench row starts only when you have candidate-note
            outputs from a real system across a declared dataset. The public artifact
            is the aggregate result: system, date, dataset, case count, judge, repeats,
            failure rates, confidence interval, and tuning disclosure.

            What goes public
            
              Aggregate scores, case count, system date, judge, repeats, confidence intervals, and method notes.
              
          
              
          
              
            What stays private
            
              Raw closed-model notes, patient data, secrets, and any candidate text you are not allowed to publish.
              
          
              
          
              
            When to submit
            
              Use 57 cases when possible, no unresolved judge errors, and clear disclosure of prompts or benchmark tuning.
