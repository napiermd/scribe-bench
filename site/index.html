ScribeBench is for clinical QA reviewers, buyers, operators, and builders who need receipts before trusting an AI-scribe claim. One note gives a QA receipt; many notes give aggregate evidence.

      What it is

      What this repo actually is

      ScribeBench is not a chatbot and not a current model leaderboard yet. It is a public QA path for one clinical note plus a reproducible harness for turning many note checks into aggregate evidence.

      Open GitHub
      System loop

      One note becomes a public card. Many notes become a public row.

      That is the repo: a walk-up QA surface, a model-backed Lab, an evaluator, and a scores-only ledger tied together by one evidence contract.

      1. 1 Bring source plus AI note

        Use a transcript, source note, demo note, or candidate-note file.

      2. 2 Check unsupported care

        The receipt names invented workups, contradictions, leaks, and boundaries.

      3. 3 Share the evidence card

        A compact artifact says what happened, what it proves, and what it does not.

      4. 4 Graduate to aggregate rows

        Run enough public cases, then publish scores only: rates, dates, judge, repeats.

              Method
              
          
              Three checks, one public scoring harness
              
          
              
            ScribeBench separates quality from safety. A note can be fluent and complete while
            still inventing something clinically meaningful, so the fabrication review is run as
            a separate adversarial judge.
          
              
        
              
        
              
          
              
            01
            
              Narrative quality
              
            
              Six physician-style dimensions: story, completeness, flow, artifacts, readability, and input fidelity.
              
          
              
          
              
            02
            
              Fabrication review
              
            
              Flags unsupported clinical content and separates dangerous invention from standard charting.
              
          
              
          
              
            03
            
              Leak detection
              
            
              Deterministic scan for raw template placeholders and internal metadata tokens.
